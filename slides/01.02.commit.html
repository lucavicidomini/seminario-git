Commit

  • Aggiungere un file (all'area di staging) vuol dire che git deve segnalare eventuali modifiche al file.
  • Per salvare in modo definitivo le modifiche al file all'interno del repository, dobbiamo fare commit delle modifiche.

Ricorda!

git memorizza modifiche, non file

Commit Cos'è

… una definizione pane e puparuoli …
Un commit è un insieme di modifiche descritte da un messaggio
  • Nota:
    • modifiche: i cambiamenti fatti sui file che abbiamo aggiunto all'indice
    • messaggio: il motivo per cui abbiamo effettuato le modifiche

Commit messageCosa scrivere?

  • Quando fate delle modifiche, dovete chiedervi:
    “Cosa sto facendo?”¹.
    • Aggiungete meta-contenuto al commit.
    • Non unite motivazioni diverse in un singolo commit.
  • Esempi:
    • Refactor class Persona as Person
    • Fix bug with Unicode characters
    • New feature: save your contacts to the cloud
  1. Intendo del codice, non della vostra vita.

Commit messageCosa non scrivere?

  • commit
  • plug-in update, bug fix, more work, minor changes, oopsie, wtf
  • Work on feature: Export as Excel (×20)
  • Fix BUG-9284
